GameSelectEvent

This event class is fired whenever the game selection in the wheel UI changes. Selection changes can occur for a variety of reasons, including simple navigation commands (Next, Previous), selecting a new filter that doesn't match the previously selected game, batch capture processing, and attract mode updates. In addition to firing when switching from one game to another, the event fires when switching to or from an empty selection (that is, no game is selected because the wheel is empty).

This event has only one subtype:

  • gameselect: Fires when a new game is selected in the wheel UI. This event fires after the fact, so it's not cancelable.

The event target for this event type is the gameList object.

Properties

This event type has all of the standard event properties and methods (see the Event class). In addition, it has the following:

  • game: A GameInfo object identifying the newly selected game, or null if no game is selected. A null selection occurs when the wheel is completely empty, which can happen when the system can't find any games to load, or when a filter is activated that doesn't have any matching games.
