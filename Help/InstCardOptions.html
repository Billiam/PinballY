Instruction Card Options

Instruction card popup location: Selects the window where the instruction card popup will appear when you use the "Instructions" command in the main menu. If the window you select is closed when you try to bring up an instruction card, the popup will appear in the main playfield window instead.

Note that this doesn't affect the separate Instruction Card window, which is designed to be positioned on a dedicated instruction card monitor on your pin cab apron. If you don't have a dedicated monitor for that, you can just close the Instruction Card window so that it doesn't take up any display space. (To close it, right-click in the window and select Close from its menu.)

Include .swf (Shockwave Flash) files when searching for instruction card images: If this box is checked, PinballY will include Shockwave Flash (.swf) files when searching for instruction card images, along with the normal set of image file types (.png, .jpg, .jpeg). If this box isn't checked, .swf files will be ignored.

PinballY can only display .swf files if you have the Adobe Shockwave Flash Player ActiveX installed on your system. If you don't have Flash installed, you'll get an error message every time PinballY tries to load an .swf file, to let you know that you need to install Flash Player to use those files. The error message is intended to be helpful, to let you know what you need to do to view those files successfully, but it's just an annoyance if you don't wish to install Flash. This option is provided so that you can get rid of those error messages if you never plan to install Flash. If you un-check this box, PinballY will silently ignore .swf files when looking for instruction card images.

By the way, if you installed a Flash plug-in for Firefox or another browser, and you're still getting error messages from PinballY saying that Flash isn't installed, the problem is that you also need the Flash plug-in for Internet Explorer (also known as the Flash ActiveX control). Different browsers use different versions of the plug-in, and PinballY specifically requires the one for IE. It doesn't matter whether you use IE as your browser or not; you still need the IE plug-in for PinballY's sake, if you want to use .swf files with PinballY.

Why is this even an issue, you might ask, when SWF is an obsolete format? Even Adobe, its creator, says it's obsolete. The reason is purely historical. HyperPin, the first front end for pin cabs, used SWF for instruction cards. That led to a large number of downloadable Media Packs for HyperPin including instruction cards in SWF format. Those old Media Packs are still around and they're still the most convenient way to get all of the media "extras" for your games. PinballY therefore supports SWF so that you can use the many existing Media Packs without having to go through any tedious format conversions every time you download one.
