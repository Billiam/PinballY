ConfigChangeEvent

This event class is fired whenever the configuration file is reloaded.

This event has only one subtype:

  • configchange: Fires after a configuration change is processed. This event fires after the fact, so it's not cancelable.

The event target for this event type is the mainWindow object.

Triggers and effects

The system automatically reloads the configuration file any time the user makes changes in the Settings dialog.

Reloading the configuration also requires reloading the game databases and game file lists, since changes to the settings can affect which files are included in the scans and which databases are loaded. Games that were previously loaded into the in-memory game list might be removed from memory as a result. Any GameInfo objects or game ID values that referred to such games will become invalid, so you could use this event to clear any global variables or other cached information you might be keeping on loaded games.

Any table media files (playfield videos or images, etc) currently being displayed are also reloaded.

Properties

This event type has all of the standard event properties and methods (see the Event class). It has no additional properties beyond the standard set.
